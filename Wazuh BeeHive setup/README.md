# SOC Homelab: Wazuh, TheHive & Custom Mimikatz Detection

This repository documents the setup and configuration of a personal Security Operations Center (SOC) lab. The goal of this project was to deploy a SIEM (Wazuh) and an Incident Response platform (TheHive), configure full endpoint telemetry archiving, and write custom detection rules to catch credential dumping attacks (Mimikatz).

##  Configuring Log Archiving (Catching Everything)

By default, Wazuh only stores logs that trigger an alert. To effectively hunt for threats and write custom rules, I needed to capture *all* endpoint telemetry.

**1. Enabling Archiving on the Manager** Edited the Wazuh Manager configuration file (`/var/ossec/etc/ossec.conf`) using `nano` to set `<logall>` and `<logall_json>` to `yes`.
![Editing ossec.conf](images/Screenshot_2026-01-26_235946.png)

**2. Configuring Filebeat** Modified the Filebeat configuration (`/etc/filebeat/filebeat.yml`) to enable the Wazuh `archives` module so the raw logs get forwarded to the indexer.
![Editing filebeat.yml](images/Screenshot_2026-01-27_000616.png)

**3. Creating the Index Pattern** In the Wazuh/OpenSearch Dashboards, I created the `wazuh-archives*` index pattern. This allows querying of all raw, unalerted log data.
![Creating wazuh-archives index](images/Screenshot_2026-01-27_000910.png)

## Adversary Emulation & Threat Hunting

**1. Executing the Attack** On the Windows 10 endpoint, I simulated an attacker attempting to dump credentials by executing `mimikatz.exe` within a PowerShell session.
![Mimikatz Execution](images/Screenshot_2026-01-26_235713.png)

**2. Hunting in the Archives** Switching back to the Wazuh Dashboard, I queried the new `wazuh-archives*` index for the string `mimi`. I successfully found the raw Sysmon process creation logs generated by the execution.
![Querying Archives](images/Screenshot_2026-01-27_001153.png)

**3. Analyzing the Artifacts** Looking closely at the parsed Sysmon event data, I identified a reliable indicator for detection: the `data.win.eventdata.originalFileName` field retained the value `mimikatz.exe`. This is a strong IOC because it can catch the execution even if the attacker renames the binary on disk.
![Sysmon Event Details](images/Screenshot_2026-01-27_001211.png)

##  Custom Detection Engineering

**1. Writing the Custom Rule** Based on the threat hunt, I wrote a custom Wazuh rule (Rule ID: `100002`) to trigger a Level 15 (Critical) alert whenever a Sysmon event contains the original filename of Mimikatz. I mapped this to MITRE ATT&CK Technique T1003 (OS Credential Dumping).

```xml
<rule id="100002" level="15">
    <if_group>sysmon_event</if_group>
    <field name="win.eventdata.originalFileName" type="pcre2">(?i)mimikatz\.exe</field>
    <description>Mimikatz detect (OriginalFileName)</description>
    <mitre>
        <id>T1003</id>
    </mitre>
</rule>
